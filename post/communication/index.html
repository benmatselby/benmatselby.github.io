Collaborative communication

For me, not only does the communication need to be electronic, it needs to be collaborative.

I have a personal preference for a communication tool in the workplace, and I will blog about that in the future (This post seemed to be a prerequisite). This is my criteria for picking a communication tool

  • Online - Town halls and phone calls are great and have their place, but for me, they are not the go to
  • Mobile friendly - It needs to be easy to use on a mobile, and not be totally inappropriate with data usage (again, another post for another day!). It needs to be easy to initiate and respond to messages. In this day and age, emoji can be the easiest and quickest way to respond üëç
  • Integration with all of the tools I use as a Software Engineer. So to name a few: GitHub; Grafana; Jenkins; If you cannot integrate these into the communication channel, it‚Äôs a red flag for me
  • Fun and engaging - I personally do not want a corporate, clinical communication tool that depresses me the moment I use it. I want fun integrations and customisations adding by the team. I want team ownership of the communication tool, so we all have a sense of control. Teams are living organisms, they have their own little personalities we should embrace that. The communication channel should reflect that
  • It has to work on a Mac and Linux environment, and be native - Not some second rate knock off. This of course expands to Windows, but generally Mac and Linux versions seem to be the cause of pain
  • Ideally it is not something so obscure it needs a cheat sheet to use. Onboarding should not be a detrimental aspect to using the tool by any member of the team.
